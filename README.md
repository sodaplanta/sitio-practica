¿Qué es GitHub Pages?
GitHub Pages es un servicio gratuito de alojamiento de sitios web estáticos directamente desde un repositorio
de GitHub. Permite publicar contenido HTML, CSS y JavaScript de forma inmediata sin configuración de
servidor, ideal para portafolios, documentación y demostraciones front-end.
Se sugio esta serie de pasos para la publicacion de la versión estática de nuestra pagina web.
1. Se creo un repositorio público en GitHub.
2. Se subieron los archivos HTML, CSS y JS de la versión estática a la rama main.
3. Se accedio a Settings > Pages en el repositorio, seleccionamos la rama fuente y se guardo.
4. GitHub genero automáticamente la URL: https://sodaplanta.github.io/sitio-practica/

Por otro lado, el sitio cuenta con limitaciones, tal como que solo soporta archivos estaticos, no tiene base de datos, ni almacenamiento persistente, el repositorio tiene que ser publico y cuenta con limites de tamaño y ancho de banda.

Se considero asimismo, Render, pero optamos por GitHub Pages por su integración directa con repositorios, despliegue sencillo y mantenimiento prácticamente nulo, ideal para sitios estáticos, mientras que Render ofrece más flexibilidad, soporte para aplicaciones dinámicas y configuraciones avanzadas, pero para nuestro caso la practicidad de GitHub Pages fue determinante, por lo que nos fuimos con esa opción.
